## SCORING METHODOLOGY

Each reviewer is scored on **two axes**:

1. **Audit Quality** — How good is their review of the site?
2. **Reviewer Performance** — How well did they execute the mission brief?

Both are scored independently. A reviewer could find genuine errors (high audit quality) but write a sloppy report (low performance), or vice versa.

---

## AXIS 1: AUDIT QUALITY (Site Score)

Each reviewer assigns the site a score out of 100, weighted:

| Criteria | Weight | Score Range |
|----------|--------|-------------|
| A. Architectural Coherence | 20% | 1–5 |
| B. Content Accuracy | 25% | 1–5 |
| C. Content Completeness | 20% | 1–5 |
| D. Cross-Referencing & Indexing | 15% | 1–5 |
| E. Translation Quality | 10% | 1–5 |
| F. User Experience & Navigation | 10% | 1–5 |
| **Weighted Total** | **100%** | **/100** |

---

## AXIS 2: REVIEWER PERFORMANCE (Professor Score)

Each reviewer's report is scored on execution quality:

| Criterion | Max Points | Description |
|-----------|-----------|-------------|
| Completeness | 25 | Covered all 6 criteria areas (A–F)? |
| Accuracy | 25 | Errors identified are genuine? No false positives? |
| Depth | 20 | Understood deeper architecture (Warp & Weft, 4Cs, Modes, Web Types)? |
| Actionability | 15 | Proposed fixes are specific and implementable? |
| Clarity | 15 | Report is well-structured and professional? |
| **Total** | **100** | |

**Bonus XP:** +10 per genuine previously-unknown error. +20 for identifying systemic error patterns.
